TimeThrottle — Full Project Breakdown

Updated: March 19, 2026

1) What the app is

TimeThrottle is an iPhone pace-analysis app built around one question:
“How much time did speed really buy you?”

It compares a user’s real or planned trip against a baseline and shows tradeoffs such as:
- time saved
- time under target pace
- fuel penalty
- ticket-risk estimate
- overall trip balance

It is positioned as a trip-analysis tool, not a built-in navigation app.

2) What the app does now

A) Live Drive
Live Drive is the real-time trip analysis mode.
It can:
- capture an Apple Maps route and ETA baseline
- use Current Location as the default route start
- offer Apple Maps-style address autocomplete for route setup
- track speed, distance, and trip progress with iPhone location services
- compare live projected pace against the Apple ETA baseline
- estimate time saved, time under target pace, fuel penalty, and trip balance
- hand off navigation to Apple Maps or Google Maps without claiming built-in navigation

Live Drive setup includes:
- From
- To
- Calculate route
- Route options
- Selected route
- Apple ETA
- Map preview
- Target speed
- MPG
- Fuel price
- Start Drive

Live Drive active screen includes:
- Live route map
- Live comparison bar
- Current speed
- Distance driven
- Time Under Target Pace
- Trip balance
- Stop Drive
- End Trip

B) Route
Route mode compares a planned trip against an Apple Maps route and ETA baseline.
It includes:
- Apple Maps route lookup
- Route options
- Route preview
- Apple ETA baseline
- Pace comparison
- Fuel and ticket-risk estimates
- Comparison bars and trip summary output

C) Manual
Manual mode compares two pace scenarios across a hand-entered distance.
It includes:
- Distance entry
- Speed A vs Speed B comparison
- Average-speed or trip-duration comparison input
- Fuel assumptions
- Pace, fuel, and ticket-risk tradeoff output

3) Major features that exist now

- Apple Maps route integration with route lookup, route options, route selection, route preview, and Apple ETA as a baseline
- Live comparison bars in Route mode and Live Drive
- Live route map in Live Drive
- “Was It Worth It?” / trip summary style outputs
- Truthful pace-based wording instead of misleading “traffic” wording
- Location permission handling for Live Drive
- External navigation handoff in v1.2
- Current Location as the default route start
- Apple Maps-style address autocomplete

4) What v1.2 added

Version 1.2 is centered on External Navigation Handoff.
It keeps Apple Maps as the planning layer and ETA baseline source, while allowing Live Drive to hand off visible navigation to:
- Apple Maps
- Google Maps
- Ask Every Time

v1.2 adds:
- provider-agnostic navigation handoff
- Apple Maps / Google Maps / Ask Every Time
- fallback if Google Maps is not installed
- Current Location default route start
- Apple Maps-style autocomplete
- cleaner Live Drive setup flow
- stronger permission/background continuity handling for believable handoff
- more consistent Live Drive / Route / Manual UI polish

v1.2 does not add built-in turn-by-turn navigation.

5) Development history / what was implemented over time

Phase 1 — Core calculator idea
- The project started as a simpler speed/time/fuel comparison app.
- Early focus: time saved, extra fuel cost, and whether driving faster was “worth it.”

Phase 2 — Real iOS app target and release pipeline
- The app became a real iPhone app with an Xcode target, signing, App Icon, screenshots, bundle ID, deployment target, privacy docs, and release prep.
- Confirmed platform direction: iOS only, iPhone only.

Phase 3 — Apple Maps route comparison
- Apple Maps route lookup, route options, route preview, and Apple ETA baseline were added.
- This made the app feel much more real-world than a pure manual calculator.

Phase 4 — UX restructuring
- Older framing was replaced by the cleaner mode-based structure:
  - Live Drive
  - Route
  - Manual
- This was a major usability improvement.

Phase 5 — Live Drive
- Real GPS tracking was added.
- Live Drive gained route selection before driving, route context preservation, live route map, live comparison bar, current speed, distance, and trip summary.

Phase 6 — Truthful wording cleanup
- Misleading “traffic” wording was replaced with pace-based wording such as “time under target pace.”
- This improved trustworthiness and product clarity.

Phase 7 — iOS-only cleanup
- Mixed-platform leftovers were cleaned up.
- The repo and project were aligned around the active iPhone app.

Phase 8 — v1.2 polish and navigation handoff
- External navigation handoff was added.
- Current Location became the default route start.
- Apple Maps-style autocomplete was added.
- Background continuity and permission behavior were made more honest and reliable.
- Multiple UI polish passes made Live Drive, Route, and Manual feel more like one product.

Phase 9 — Repo/folder cleanup
- Generated output and local machine state were moved out of version control using ignore rules.
- Legacy resources and the stale macOS folder were removed.
- The repo now reflects the real iPhone app more cleanly.

7) Important files and what they do

- TimeThrottleApp_iOS.swift
  App entry point for iOS.

- IOSRouteComparisonScreen.swift
  Hosts the main screen flow and starting mode/layout.

- RouteComparisonView.swift
  Main UI brain of the app.
  Handles:
  - Live Drive UI
  - Route mode UI
  - Manual mode UI
  - route setup
  - map preview
  - comparison bars
  - result cards
  - Start/Stop/End Drive flow
  - summary cards and labels

- RoutePreviewMapView_iOS.swift
  Displays iOS route previews and active route cards.

- TimeThrottleCalculator.swift
  Core trip math, especially segment/manual-style calculations.
  Uses truthful naming such as timeUnderTargetPaceMinutes.

- SpeedCostCalculator.swift
  Speed/cost comparison math such as fuel penalty and trip summary components.

- LiveDriveTracker.swift
  Core Live Drive tracking service.
  Handles:
  - Core Location
  - permission state
  - trip start / stop / reset
  - current speed
  - distance traveled
  - incremental updates
  - inputs for trip summary calculations

- TripAnalysisEngine.swift
  Shared live trip analysis layer.
  Interprets progress against baseline and produces TripSummary-style results.

- RouteModels.swift
  Shared route-related models and state.
  Handles:
  - route options
  - selected route
  - baseline route context
  - mode configuration
  - autocomplete / endpoint state

- NavigationHandoffService.swift
  Handles Apple Maps / Google Maps / Ask Every Time handoff behavior added in v1.2.

- No advertising SDK is included
