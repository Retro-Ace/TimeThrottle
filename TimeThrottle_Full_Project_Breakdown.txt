TimeThrottle — Full Project Breakdown (v2.0)

1. WHAT THE APP IS

TimeThrottle is an iPhone Live Drive pace-analysis app with a separate public scanner listening tab, built around one simple driving question:

"How much time did speed really buy you?"

It is not an Apple Maps replacement or Apple-native navigation engine.

It uses Apple Maps as the route lookup and ETA-baseline layer, then tracks a real drive to show:
- Time Above Speed Limit
- Time Below Speed Limit
- projected arrival versus the Apple Maps ETA baseline
- guidance based on Apple Maps route steps

In Live Drive, the app can optionally hand off visible navigation to Apple Maps, Google Maps, or Waze while TimeThrottle keeps doing the trip tracking and pace analysis.

2. WHAT THE APP DOES NOW

As of v2.0, the app is centered around:
1) Live Drive with route intelligence
2) Scanner for informational public scanner listening

A. Live Drive
This is the real-time driving mode and the app’s full product focus.

It lets the user:
- choose a route baseline from Apple Maps
- start from Current Location by default
- use Apple Maps-style address autocomplete
- calculate route options
- choose a route
- preview the route before starting
- start a live tracked trip
- see Time Above Speed Limit / Time Below Speed Limit during the drive
- see route-step guidance during the drive
- review route weather checkpoints
- see route weather checkpoint markers on the active route when forecast data is available
- see cached OpenStreetMap speed-limit estimates where available
- optionally show passive Nearby Low Aircraft from ADS-B data with conservative refresh, approximate marker projection, optional audio cues, and stale cleanup
- see a compact nearest-aircraft Map bar when nearby aircraft data is available
- optionally show passive Enforcement Alerts from the configured OpenStreetMap Overpass camera/enforcement source, capped at 30 visible reports within 3.0 miles, with quiet empty states when no tagged source data is available
- switch Standard / Satellite map mode
- use a visible floating Recenter control on Map
- see a compact floating weather chip on Map when route forecast data is available
- save Top speed from valid GPS speed samples on completed trips
- pause, resume, and end the trip
- save the finished trip
- share a finished-trip summary
- reopen past trips in Trip History
- switch between Drive, Map, Trips, and Scanner using bottom navigation

B. Scanner
Scanner is separate from Live Drive.

It lets the user:
- browse public scanner systems
- use Nearby scanner discovery when location is available
- search by system name, short name, city, county, or state
- open a selected scanner system
- use Live Feed playback when a permitted direct stream URL is configured
- view latest public scanner calls
- play the selected call or latest playable call, pause, and advance scanner audio
- continue scanner playback in the background after starting audio

Scanner uses an OpenMHz-style API client with a configurable base URL for Latest Calls and a bundled approved-direct-stream config for optional Live Feed playback. It does not scrape Broadcastify, does not record scanner audio, does not upload scanner feeds, does not support user-uploaded scanner feeds, and does not feed Live Drive calculations, route intelligence, route warnings, incident prediction, or driving recommendations. Live Feed and Latest Calls replay are separate playback modes.

Live Drive setup includes:
- compact automatic Current Location status
- To
- Route options
- Selected route
- Apple ETA
- Map preview
- Start Drive

Bottom navigation includes:
- Drive: setup, Current Location start, destination input, route options, compact navigation app choice, and Start Drive
- Map: primary active driving HUD with route polyline, route weather checkpoint markers when available, user location, next maneuver, speed, Speed Limit estimate where available, Apple Maps ETA, projected arrival, route distance, miles driven, Pause / Resume, End Trip, recenter, and Options
- Trips: Trip History list and trip details
- Scanner: Nearby / Browse public scanner systems, optional configured Live Feed playback, selected-system latest calls, and replay

Live Drive while active includes:
- current speed
- elapsed trip time / trip status
- Apple Maps ETA
- projected arrival / live arrival context
- Time Above Speed Limit
- Time Below Speed Limit
- distance
- Pause / Resume
- End Trip
- next maneuver and distance
- voice guidance mute / unmute
- voice choice, test prompt, and speech-speed controls in the Route Info drawer and Map Options panel
- off-route and reroute status
- Speed Limit estimate / Unavailable state
- route weather status
- optional Nearby Low Aircraft status, projected markers, and audio cue control
- optional Enforcement Alerts status, active HUD warning, marker layer, red-light camera filter, enforcement-report filter, and camera audio cue control when enabled and available
- Map Options details for Map Mode, compact weather, aircraft, voice, speed limit, pace, and Enforcement Alerts
- route/map context
- optional external navigation handoff

Live Drive finished-trip flow includes:
- result-first summary
- overall result / verdict
- key trip stats
- shareable summary
- Trip History save path

4. CURRENT WORDING / PRODUCT FRAMING

The app now uses direct pace-based wording.

Examples:
- Time Above Speed Limit
- Time Below Speed Limit
- Apple Maps ETA
- projected arrival

The app remains careful not to claim:
- Apple-native turn-by-turn navigation
- lane guidance
- certified speed-limit accuracy
- live traffic intelligence it does not truly own
- aviation safety or collision-avoidance alerts
- certain enforcement detection or legal guarantees
- scanner recording
- scanner-based driving recommendations
